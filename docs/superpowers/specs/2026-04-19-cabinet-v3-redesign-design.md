# Cabinet v3 — Редизайн и платформенная стратегия

**Дата**: 2026-04-19
**Статус**: Design approved, готов к implementation plan
**Исходный запрос**: Кнопка Bitrix24 в кабинете + редизайн в «блочном/не-шаблонном» стиле

---

## 1. Контекст и цели

### Стратегическое направление
Веб-кабинет становится **основной платформой** Tender Sniper. Telegram и VK Max — каналы получения уведомлений и быстрых действий, не основные рабочие места. Это формулировка пользователя, зафиксирована в `memory/project_platform_strategy.md`.

Следствия:
- Любая новая фича проектируется сначала для кабинета
- Со временем TG/Max упрощаются до уведомлений + базовых действий
- Cabinet v3 должен быть самодостаточным

### Почему редизайн именно сейчас
- Текущий кабинет v2 (deployed 2026-03-24) выглядит как «lite-dashboard» — не передаёт ощущение платформы
- Правая часть экрана визуально пустая, текст сложно читать
- В roadmap v2 — собственный Kanban pipeline (замена Bitrix24 как CRM) — раскладку под него надо закладывать сразу

### Цели v3
1. **Не-шаблонный облик** — отход от generic «AI dashboard» карточек
2. **Лёгкое чтение при долгом использовании** — строгие шрифты, спокойная палитра
3. **Подготовка к Kanban (v2 roadmap)** — архитектура раскладки должна вмещать доску
4. **Все 11 страниц в единой системе** — стиль-гайд, применимый ко всему кабинету

### Non-goals
- Менять бэкенд (aiohttp + серверный рендер остаются)
- Добавлять JS-фреймворк (vanilla + лёгкие утилиты)
- Делать тёмную тему (только светлая на MVP, toggle добавим позже если понадобится)
- Реализовывать Kanban в этом спеке — он в v2
- Менять доменную логику фильтров, GPT, подписок, платежей
- Mobile-first — desktop-first, адаптив добавим позже

## 2. Scope

### In-scope — Cabinet v3 (этот спек)
Редизайн **всех 11 страниц** кабинета под единую систему:

| Страница | Путь | Ключевой паттерн |

| Tender-GPT | `/cabinet/gpt` | Chat-layout |
| Фильтры | `/cabinet/filters` | CRUD list + карточки |
| Документы | `/cabinet/documents` | List с превью/скачиванием |

| Профиль | `/cabinet/profile` | Форма компании |

| Подписка | `/cabinet/subscription` | Pricing-cards + billing |

### Out-of-scope — отложено на v2 и далее
| Что | Когда | Примечание |

| Multi-user + роли | v3+ | Требует redo БД |
| Email-дайджест | v3+ | Дешёвая фича, но не критично |
| Excel-экспорт подборок | v3+ | Можно сделать быстро параллельно |
| Проверка аффилированности | v3+ | Killer-фича, нужен парсинг протоколов |
| Превью документов без скачивания | v3+ | UX-бонус |
| Поиск по содержимому тендерной документации | v3+ | Тяжёлая индексация |
| Diff изменений в избранных | v3+ | Retention-хук |

### Bitrix24 в кабинете — минимальная интеграция
- Кнопка «В Битрикс24» появляется в модалке тендера (рядом с «В Pipeline»)
- Создаёт базовую сделку: TITLE, OPPORTUNITY, COMMENTS (вся мета в текст)
- Без расширенного маппинга (UF_CRM_* полей, стадий, ответственных) — оставляем только у Николая через флаг
- Для универсальных юзеров — стандартная сделка, стандартная стадия
- Webhook настраивается в `/cabinet/settings` → «Интеграции»

## 3. Дизайн-система

### 3.1 Цветовая палитра (кремовая + жжёная сиена)

Подход: тёплая «бумажная» основа, сиена как единственный акцент.

| Токен | Значение | Применение |
|---|---|---|
| `--bg` | `#f5f2ea` | Фон main-зоны |
| `--bg-2` | `#efeadf` | Фон сайдбара, правой колонки |
| `--bg-3` | `#faf7f0` | Поднятые блоки (quota, help) |
| `--bg-raised` | `#ffffff` | Карточки, формы, модалки |
| `--line` | `rgba(30,25,18,0.10)` | Границы |
| `--line-soft` | `rgba(30,25,18,0.06)` | Разделители внутри списков |
| `--text` | `#1a1814` | Основной текст |
| `--muted` | `#8a8474` | Лейблы, вторичная инфа |
| `--sub` | `#52493c` | Подтекст |
| `--accent` | `#a05a1e` | Акцент (сиена) |
| `--accent-dim` | `rgba(160,90,30,0.08)` | Фон акцентных блоков |
| `--accent-line` | `rgba(160,90,30,0.25)` | Границы акцентных блоков |
| `--alert` | `#b13d28` | Горящие сроки, ошибки |
| `--positive` | `#567a3f` | Успех, зелёные метрики |

### 3.2 Типографика

| Шрифт | Назначение | Где |
|---|---|---|
| `Inter` (основной) / `Manrope` (fallback) | Текст, заголовки, кнопки | Всё UI-содержимое |
| `IBM Plex Mono` | Данные, номера, метки, даты, суммы | Всё табулярное и техническое |
| `Unbounded` (weight 900) | Логотип | Только `.logo` |

**Никаких серифов и италиков в контенте.** Строгие sans + mono.

**Размеры:**
- Page h1: 30px / 600 / tracking -0.025em
- Section h3: 13-14px / 600
- Body: 14px / 400-500 / line-height 1.6
- Meta/labels: 11-12px / 500, часто в IBM Plex Mono
- Micro-labels (ALL-CAPS): 10px / mono / letter-spacing 0.14em / text-transform uppercase
- Ценовые «герои»: 22-30px / mono / 600 / tracking -0.01em

**Особенности:**
- `font-feature-settings: "ss01", "cv11"` — включаем альтернативную «1», «а», «l» из Inter
- `font-variant-numeric: tabular-nums` — цифры выравниваются в колонку

### 3.3 Сетка и пространство

- Базовая сетка: 4px baseline
- Раскладка: 220px (sidebar) + fluid main + 300px (right rail)
- Main padding: `36px 44px`
- Right rail padding: `28px 20px`
- Sidebar padding: `24px` top, `14-20px` sides
- Max-width main: нет — заполняет оставшееся пространство

### 3.4 Раскладка приложения

3-колоночная архитектура для большинства страниц:
```
┌──────────┬──────────────────────────┬────────────┐
│          │                          │            │
│ Sidebar  │      Main content        │ Right rail │
│  220px   │       (flex)             │   300px    │
│          │                          │            │
└──────────┴──────────────────────────┴────────────┘
```

Адаптивные правила:
- `≥1280px`: все три колонки
- `960-1279px`: sidebar + main, right rail скрыта (можно вытащить кнопкой)
- `<960px`: sidebar как drawer, full-width main

Kanban-страница (v2) получит 2-колоночный режим (sidebar + full-width main), без right rail.

## 4. Компоненты

### 4.1 Sidebar

- Логотип сверху (`font-family: Unbounded`)
- Группы навигации, разделённые `.sep` (1px линия, 10px margin):
  - **Работа**: Тендеры, Pipeline (v2), Календарь, Документы
  - **Источники**: Фильтры, Поиск, Tender-GPT
  - **Данные**: Статистика, Профиль, Настройки
- Счётчик справа в каждой ссылке (`font-family: mono`)
- Активный пункт — `background: --accent-dim`, `color: --accent`
- Внизу — блок пользователя (аватар 32px + имя + тариф)

### 4.2 Page Header

```
┌────────────────────────────────────────┐
│ МОНО-НАДЗАГОЛОВОК (дата или категория) │
│ H1 — с акцентной подсветкой (ключ)    │
│ Сводка — 1-2 строки с ключевыми числ. │
└────────────────────────────────────────┘
```

### 4.3 Tender Row (feed-строка)

```
┌─────────────────────────────────┬───────┐
│ ЗАГОЛОВОК ТЕНДЕРА               │ 4,2 M │
│ заказчик · закон · срок         │ млн ₽ │
│                                 │ 94%   │
└─────────────────────────────────┴───────┘
```
- Hover: лёгкий сдвиг + фон `rgba(160,90,30,0.025)`
- Клик → открывает модалку
- Нет нумерации (001/002 убрали — читались как баг-трекер)

### 4.4 Modal (детальная карточка тендера)

- Размер: `max-width: 780px`, `max-height: calc(100vh - 80px)`
- Backdrop: `rgba(20,15,10,0.45)` + `backdrop-filter: blur(2px)`
- Сетка модалки:
  - Header: название + сумма справа + кнопка закрытия
  - Body (2 колонки): слева Основное + Документы, справа AI + История
  - Footer: вторичные кнопки + primary CTA справа
- Закрытие: `ESC`, клик на backdrop, крестик

### 4.5 Chips (фильтры/вкладки)

- Размер: 13px текст, 7px/14px padding, border-radius 100px
- Default: прозрачный фон, `--line` граница, `--sub` текст
- Hover: `--text` color
- Active: `--accent-dim` фон, `--accent-line` граница, `--accent` текст
- Счётчик справа: `font-family: mono`, 11px, `--muted`

### 4.6 Buttons

**Primary (CTA)**
- Фон `--accent`, текст `#fff`, padding 10-12px × 16-18px
- Hover: `#b4692a` (чуть светлее)
- Border-radius: 3px

**Secondary**
- Фон `--bg-raised`, граница `--line`, текст `--sub`
- Hover: `--text` color, `--line` → тёмнее

**Icon button**
- 34×34px квадрат, граница `--line`, символ в mono
- Используется для ✎ (edit), ⋯ (more), ✕ (close)

### 4.7 Toggle (on/off)

- 42×22px pill, `rgba(30,25,18,0.15)` фон
- Шарик 18×18, `--bg-raised`, маленькая тень
- Active: фон `--accent`

### 4.8 Quota-бар (использование лимитов тарифа)

- Для лимитов: фильтры, GPT-сообщения, уведомления
- Фон `--bg-3`, граница `--line`, padding 16-20px
- Слева текст «5 из 20», справа прогресс-бар 180px + «5 / 20» моно под ним

### 4.9 Right rail — блоки

Стандартные блоки для сквозного использования:

**Pipeline preview**
```
Найдено    23
Изучаем     6
Заявка      3
Подано      2
Результат   0
```
Строки — `bg-raised` + `line` границы, 8-10px padding.

**Ближайшие сроки**
- Левая колонка: дата (моно) + дней до
- Правая колонка: заголовок + заказчик
- 4 ближайших срока + кнопка «Календарь»

**AI-подборка**
- `bg-raised` с левой границей `--accent`, 3px
- Бейдж «Шанс победы 72%» сверху
- Заголовок тендера + 1 строка мета

**Топ-фильтр недели**
- Для страницы фильтров: конверсия, сумма

**Советы**
- Для страницы фильтров: 3 карточки с советами

### 4.10 AI-блок

Единый паттерн для всех AI-сводок:
```
╔═══════════════════════════════════╗
║ [accent-left-border 3px]          ║
║ Текст AI-анализа с выделениями.   ║
║                                   ║
║ Шанс победы  [━━━━━━━ ░░░░]  72% ║
╚═══════════════════════════════════╝
```

## 5. Паттерны страниц

### 5.1 Тендеры (главная) — `/cabinet/`
- Header: «Сегодня — N новых тендеров» + дата сверху
- Саммари: горящие / в работе в одной строке
- Chips: «Все / Сегодня / Горящие / В работе / По фильтру»
- Feed: строки (см. 4.3)
- Пагинация внизу (моно)
- Right rail: Pipeline preview + Ближайшие сроки + AI-подборка

### 5.2 Фильтры — `/cabinet/filters`
- Header: «Фильтры — N активных» + primary CTA «+ Новый фильтр»
- Quota-бар
- Tabs: Все / Активные / Выключенные
- Карточки-строки фильтров: название, ключевые слова (чипы), мета (бюджет/регионы/закон), счётчик совпадений справа, toggle + actions
- Right rail: Топ-фильтр недели + Советы
- Редактирование — в модалке (новая или ✎)

### 5.3 Pipeline — `/cabinet/pipeline` (v2, не этот спек)
Важно зарезервировать путь в навигации и архитектуре сейчас, чтобы в v2 просто подключить.

### 5.4 Календарь — `/cabinet/calendar`
- Месячная сетка 7×6
- Активные дни — с акцентом, подача/результат — разные цвета
- Клик на день — right rail показывает тендеры этого дня
- Переключение view: Месяц / Неделя / Список

### 5.5 Документы — `/cabinet/documents`
- Список документов, сгенерированных AI или скачанных
- Фильтр по типу (pdf/docx/xlsx) + по тендеру
- Превью без скачивания — v3+, пока скачивание

### 5.6 Поиск — `/cabinet/search`
- Форма поиска вверху (как chip-row)
- Результаты — те же feed-строки, что на главной
- Правая колонка: сохранённые поиски (будущее)

### 5.7 Tender-GPT — `/cabinet/gpt`
- Отдельная раскладка: sidebar + chat + right rail с контекстом сессии
- Chat-пузыри: пользователь (обычный background), GPT (левая accent-граница)
- Ввод внизу с иконками приложений (тендер, документ)

### 5.8 Статистика — `/cabinet/stats`
- 4 основные метрики вверху (квадраты со статами)
- Графики: conversion, win-rate, суммы в работе по времени
- Right rail: экспорт, настройки периода

### 5.9 Настройки — `/cabinet/settings`
- Секции: Уведомления, Email, Telegram, Max, **Интеграции** (Bitrix24), Безопасность, Удаление аккаунта
- Каждая секция — белая карточка, заголовок + форма
- Bitrix24: поле webhook URL + кнопка «Проверить соединение» (вызов `validate_bitrix24_webhook`) + toggle включения
- Валидация через `validate_bitrix24_webhook` из `bot/handlers/bitrix24.py`

### 5.10 Профиль — `/cabinet/profile`
- Данные компании: название, ИНН, КПП, банк, контакты, описание ниши
- Используется для Bitrix24 и документов (шапки заявок)
- Форма с секциями

### 5.11 Подписка — `/cabinet/subscription`
- Текущий тариф: карточка с датой окончания + кнопка «Продлить»
- 3 тарифа: Starter / Pro / Business — cards с ценой и фичами
- История платежей

### 5.12 Логин — `/cabinet/login`
- Минимальная страница: логотип + кнопка «Войти через Telegram»
- Центрирована, без сайдбара

## 6. Техническая архитектура

### 6.1 Файловая структура
```
cabinet/
├── templates/
│   ├── _base.html         (единый базовый layout)
│   ├── _sidebar.html       (партиал сайдбара)
│   ├── _rightrail.html     (партиал правой колонки)
│   ├── _modal_tender.html  (модалка тендера)
│   ├── dashboard.html      (экстендит _base)
│   ├── filters.html
│   ├── ... (все страницы)
│   └── login.html
├── static/
│   ├── css/
│   │   ├── tokens.css      (CSS-переменные)
│   │   ├── base.css        (reset + body + typo)
│   │   ├── components.css  (sidebar, chips, buttons, toggle, modal, etc.)
│   │   └── pages/
│   │       ├── dashboard.css
│   │       ├── filters.css
│   │       └── ... (per-page overrides)
│   ├── js/
│   │   ├── modal.js        (open/close + ESC + backdrop)
│   │   ├── toggle.js
│   │   └── api.js          (fetch-обёртки)
│   └── fonts/              (если self-host)
├── routes.py
├── api.py
└── auth.py
```

### 6.2 Загрузка шрифтов
- Через Google Fonts CDN на MVP: Inter, IBM Plex Mono, Unbounded
- `preconnect` + `display=swap` (уже делаем в текущем кабинете)
- Self-host опционально потом, если появится скорость-регрессия

### 6.3 JS-минимум (vanilla)
Нужно сделать:
- `modal.js`: open/close, ESC, click-backdrop, фокус-trap
- `toggle.js`: click по toggle → fetch API → обновить UI
- Общий `api.js`: обёртка над fetch с CSRF + обработкой ошибок
- Chip-фильтры: изменение URL `?filter=horячий` без reload
- Sorting dropdown: тоже через URL
- Никаких React/Vue/Alpine — оставляем серверный рендер + progressive enhancement

### 6.4 Бэкенд — изменения
- Routes в `routes.py` остаются
- Новые шаблоны в `templates/` (перевёрстка существующих)
- Partial-рендеринг через Jinja2 (уже используется через aiohttp-jinja2)
- API уже готово (`api.py`) — редизайн — это только презентационный слой

### 6.5 Bitrix24 минимальный flow
- `cabinet/settings` → секция «Интеграции» → поле webhook
- API: `POST /cabinet/api/settings/bitrix24` — валидирует + сохраняет в user.data
- API: `POST /cabinet/api/settings/bitrix24/test` — использует `validate_bitrix24_webhook`
- На модалке тендера: кнопка «В Битрикс24» → `POST /cabinet/api/tenders/{n}/bitrix24`
- В `bot/handlers/bitrix24.py` добавить функцию `create_simple_bitrix24_deal` (только TITLE, OPPORTUNITY, COMMENTS) — для не-расширенных юзеров
- Расширенный режим (для Николая, `user.data['bitrix24_extended']=True`) — вызывает текущую `create_bitrix24_deal` со всеми UF_ полями

## 7. Migration plan (в порядке реализации)

1. **Токены + base.css** — CSS-переменные, шрифты, reset, базовая типографика
2. **Компоненты** — sidebar, chips, buttons, toggle, tabs, quota-bar, modal, rr-blocks
3. **Главная (dashboard)** — первая страница в новом стиле
4. **Модалка тендера** — универсальная
5. **Фильтры** — валидирует формы
6. **Поиск** — тот же feed-паттерн
7. **Tender-GPT** — chat-layout
8. **Профиль + Настройки** — формы, интеграции, Bitrix24
